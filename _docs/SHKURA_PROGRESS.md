# SHKURA Visual Editor: Прогресс жана Колдонмо

> [!IMPORTANT]
> **ЖАҢЫ АССИСТЕНТТЕР ҮЧҮН МААНИЛҮҮ ЭСКЕРТҮҮ:**
> Бул файл - Shkura Visual Editor'дун учурдагы абалынын "күндөлүгү". Ишти баштоодон мурун ушул файлды окуп чыгыңыз. Бул жерде ишке ашырылган архитектуралык чечимдер жана кийинки кадамдар жазылган. Эч качан "старт" протоколун бузбаңыз жана планыңызды колдонуучу менен макулдашыңыз.

## Бүгүнкү күнгө карата ишке ашкан функциялар (Phase 1-5)

### 1. Тексттик элементтер жана Дизайн
- Текст кошуу, өлчөмүн, шрифтин (`fontFamily`), калыңдыгын (`fontWeight`) жана түсүн өзгөртүү.
- Текстти түздөө (`textAlign`) жана динамикалык көлөкө (`textShadow`) берүү.
- Элементтерди дублировать (📋) кылуу — бардык параметрлери (стиль, анимация, sticky) менен толук көчүрмө түзүү.

### 2. Интерактивдүүлүк (Portals)
- 3-абалдуу (`interactionType`): 
  - `none`: Анимациясыз, басылбайт.
  - `modal`: Маалыматтык терезени ачат.
  - `link`: Башка порталга же тышкы шилтемеге багыттайт (`window.location` же `window.open`).

### 3. Анимация Мотору (Advanced Motor)
- `speed`, `amplitude`, `noise` параметрлери аркылуу ар бир элементтин "дем алуу" (idle) анимациясын башкаруу.
- `Noise` эффекти элементтерге башаламандык берип, аларды "тирүү" көрсөтөт.

### 4. Sticky / Parent-Child синхрондоштуруу
- Элементтерди бири-бирине байлоо (`parentId`).
- **Organic Lag:** Бала-элементтер атасын ээрчигенде `spring` физикасын жана тандалган `stickyLag` (0.1с — 1.0с) кечигүүсүн колдонушат.
- **Idle Sync:** Бала-элементтер атасынын анимация ритмин (speed/noise) автоматтык түрдө синхрондуу кайталайт.

### 5. Туруктуулук жана Визуалдык Тегиздөө (Rendering Stability)
- **Anchor Migration:** Позициялоо (static div) жана анимация (motion.div) катмарлары экиге бөлүндү. Бул Framer Motion тарабынан `transform` касиеттеринин өчүрүлүп кетишин (перезапись) токтотту.
- **Visual Sync:** Тексттер жана кастом иконкалар эми дайыма өз рамкасынын (hitbox) ичинде 100% тактык менен турат. 
- **Full Layer Logic:** ИД 1-5 катмарлары үчүн `0:0` (жогорку-сол) бекем орнотулду. Сүрөт алмаштырылса дагы алар бүтүн экранды каптап турат.

## Техникалык архитектура

- **State Management:** `EditorContext.tsx` — бардык убактылуу өзгөрүүлөр (`tempPositions`, `tempStyles`, `tempAnimations`, ж.б.) ушул жерде сакталат.
- **Rendering:** `MapIcon.tsx` — **Эки кабаттуу контейнер** аркылуу (positioning + animation) физика жана анимациялар ишке ашат.
- **Persistence:** `saveLocations` функциясы (`locations.ts` файлына жазат).
