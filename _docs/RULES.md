# Jer Satyp Aluu — Долбоор Боюнча Негизги Эреже жана Маалымат

Бул папка — долбоордун **"жады"**. Кийинки ассистенттер же иштеп чыгуучулар ишти улантмак болсо, **алгач ушул файлды окуу керек**.

---

## 🧠 Долбоор деген эмне?
`Jer Satyp Aluu` — Кыргызстандын оюн долбоору. Next.js + TypeScript + Tailwind CSS + Framer Motion менен жасалган.

## 📦 Архитектура Принципи: "Коробка Системасы"

Долбоор **"Коробка" (Feature-based)** принципи менен курулат:
- Ар бир өзүнчө функция (оюн картасы, квесттер, баатырлар) — **өзүнчө коробка** (`src/features/...`).
- Коробкалар **"жип"** (shared types/interfaces) аркылуу гана туташат.
- Жиптин ортосунда **"кулпу"** бар — TypeScript интерфейстери. Бул кокустан ката кетирүүдөн сактайт.

```
src/
├── features/           # Коробкалар
│   └── main-menu/      # Окурду баштагы мессе
│       ├── components/ # Дизайн жана Сөөк
│       ├── constants/  # Маалымат (coordinates, config)
│       ├── hooks/      # Мээ (логика, states)
│       ├── types/      # Кулпу (TypeScript)
│       └── MainMenuView.tsx
├── shared/             # Жалпы ресурстар
└── app/
    └── page.tsx        # Болгону коробканы чакырат
```

---

## ⚠️ Катуу Эрежелер (МААНИЛҮҮ)

1.  **`page.tsx` ичине логика жазма.** Ал болгону `<SomeFeatureView />` чакырат.
2.  **Коробканы сырттан түздөн-түз өзгөртпө.** Тышкы модулдар болгону экспорттолгон API аркылуу туташат.
3.  **Координаттарды (иконка позицияларын) түздөн-түз компонентке жазма.** Аларды `constants/locations.ts` файлына гана жаз.
4.  **`object-fit: cover/contain` колдонба.** Бардык сүрөттөр `position:absolute; width:100%; height:100%;` менен жайгашат (Оюн Движок ыкмасы).
5.  **Tailwind классынан тышкары инлайн стиль жазса болот**, айрыкча layout жана позиция үчүн.

## 🎨 Visual Editor (Шкура) Системасы

Шкура — UI кабатынын үстүнө кийгизилген редактор. Ал `src/features/visual-editor/` папкасында жайгашат.
Толук спецификация: `_docs/visual-editor/SPEC.md`

**Шкура Эрежелери:**

6.  **Шкура Core Logic'ке (hooks, API, state) эч качан тийбейт.** Ал болгону CSS/HTML позицияларды гана өзгөртөт.
7.  **Редактор режими күйгөндө (`isEditorMode = true`) — бардык Framer Motion анимациялары токтойт.** Антпесе элементтер учуп кетет.
8.  **SAVE басмайынча эч кандай өзгөртүү файлга жазылбайт.** Убактылуу абал болгону браузерде сакталат.
9.  **Шкура `constants/locations.ts` гана жаңыртат** — башка файлдарга жазбайт (Фаза 1).
10. **Шкура Фаза 1 (Модулдук Архитектура) аяктагандан кийин гана жазылат.**
11. **"Айнек катмарлар" (Hitboxes) физикалык жактан иконканын өлчөмүнө дал келиши керек.** Ашыкча чоңдукка жол берилбейт, баары тыкан жана өз ордунда турушу шарт.
12. **[КАТААЛ ЭРЕЖЕ] 3-Катмарлуу Координата Системасы (VISUAL OFFSET):** Шкура редактору `locations.ts` ичинде 3 көз карандысыз координата менен иштейт:
    - `x` жана `y`: Бул чыныгы **Көк Рамка (Hitbox)** басуу чекитинин координаттары. Бул оюн логикасы үчүн эң маанилүүсү.
    - `handleX` жана `handleY`: Бул **Кызыл Рамка (Офсет туткасы/Handle)** координаты. Бул редактордо элементти кармоо/сүйрөө үчүн гана керек.
    - `visualX` жана `visualY`: Бул **Сүрөттүн өзүнүн (Render Image)** абсолюттук координаты. Бул маптын башкы контейнеринен (0:0) эмес, **Hitbox'тун жогорку-сол бурчунан (top-left)** эсептелет!
    *Эскертүү: Бул 3 координатаны бириктирүүгө болбойт. Алар бири-биринен 100% ажыратылган.*

13. **[КАТААЛ ЭРЕЖЕ 🚨] Анимация жана CSS Transform Сүрүлүшү (Framer Motion Bug):** 
    - `framer-motion` китепканасы `animate={{ y, scale }}` иштеткенде, элементтеги **баштапкы CSS `transform` (мисалы `translate(-50%, -50%)`) буйруктарын өчүрүп салат**.
    - **Чечим:** Визуалдык тексттер жана кастом сүрөттөр эч качан коддо `translate(-50%, -50%)` менен борборлоштурулбашы керек. Алар дайыма **жогорку-сол бурчтан (`transform: none`)** тартылат. Пайдалануучу редактор аркылуу визуалдык катмарды (`visualX/Y`) өзү кол менен тууралап (подгонка) коёт. Подгонка бузулбашы үчүн код мээсине тийүүгө тыюу салынат!

14. **[КАТААЛ ЭРЕЖЕ 🚨] Background Layers vs Custom Elements:** 
    - **ИД 1ден 5ке чейинки катмарлар (Фондор):** Булар ДАЙЫМА `0:0` координатында, өлчөмү `100vw x 100vh` катары, `object-fit: fill` менен жайгашат. (Колдонуучу pechat.png сыяктуу тунук 1920x1080 сүрөт койгон күндө дагы, ал сүрөт кичирейбеши керек).
    - **Башка элементтер (ИД 6+):** Алар Hitbox өлчөмүнө же өздөрүнүн `auto` өлчөмүнө таянышат жана эч качан 100vw болуп чоюлбайт.

15. **[КАТААЛ ЭРЕЖЕ] Маалыматтардын бүтүндүгү (Data Integrity):** Коддогу демейки маанилер (default values) эч качан `locations.ts` файлындагы даяр маалыматтардан жогору турбашы керек. Эгер файлда маани (мисалы, `visualX`, `visualY`) бар болсо, аны демейки маани (мисалы, `0%` же `50%`) менен алмаштырууга жол берилбейт. Базадагы маани дайым УТУП ЧЫГАТ.

16. **[🛡️ КАТЫРЫЛГАН] Анимация жана Target Safety:** Редактордун анимация системасы (`AnimatedContainer.tsx`) жана таргетинг режимдери (ELEMENT, HITBOX, VISUAL) **блокко** салынган. Бул архитектураны өзгөртүүгө же "оптимизациялоо" кылууга тыюу салынат. Толук эрежелер: `_docs/visual-editor/TARGET_MODES.md`. Ар бир ассистент ишти баштоодон мурун бул документти окушу ШАРТ.

17. **[🛡️ КАТЫРЫЛГАН] Layer UI Integrity and Group Expansion:** 
    - Группалардын (Folders) ичиндеги элементтер сырткы элементтердей эле толук интерактивдүү болушу шарт (Eye, Pencil, Duplicate, Trash баскычтары жеткиликтүү).
    - Катмарлардын номерлөөсү дайыма 1-ден баштап (`layerOrder` индекси + 1) динамикалык түрдө эсептелиши керек. `#` префиксин колдонууга тыюу салынат.
    - Группанын контейнери (Folder Wrapper) эч качан `overflow: hidden` болбошу керек. Антпесе группа ачылганда ички элементтер "кесилип" (clipping) көрүнбөй калат. Дайыма `overflow: visible` (же демейки абалды) сактоо керек.

## 🎓 Техникалык Сабактар (Lessons Learned)

Бул бөлүм Шкура жана Фондук редакторду иштеп чыгууда келип чыккан мүчүлүштүктөрдүн негизинде түзүлдү. Кийинки ассистенттер үчүн колдонмо:

1.  **Фондун Дареги (Background Paths):**
    - **Маселе:** Колдонуучу же система `config.ts` файлына `"fon.png"` сыяктуу кыска ат жазса, браузер аны иштетпей "кара экран" берип коюшу мүмкүн.
    - **Чечим:** Ар дайым `MapBackground.tsx` же окшош компоненттерде даректи тууралоо логикасын (`prefix override`) иштетиңиз. Эгер дарек `/` же `http` менен башталбаса, ага автоматтык түрдө `/assets/main_menu/` префиксин кошуу керек.

2.  **Сактоо Механизми (Global Save):**
    - **Маселе:** Демейки `SAVE` кнопкасы ГАНА `locations.ts` файлын жаңыртчу. Фондун (Background) жөндөөлөрү сакталбай калып жаткан.
    - **Чечим:** Сактоо механизми (`Toolbar.tsx` -> `codeWriter.ts` -> `route.ts`) эки тараптуу болушу шарт: бир жагынан элементтерди, экинчи жагынан глобалдык сценарий жөндөөлөрүн (`config.ts`) кошо жазып турушу керек.

3.  **Оптимизация (.webp vs .png):**
    - **Маселе:** Долбоордо чоң сөлөкөттөр жана фондор `.png` түрүндө көп орун ээлейт.
    - **Чечим:** Жаңы кошулган ресурстарды `.webp` форматында сактоого артыкчылык бериңиз. Бирок коддо ар дайым `.png` же башка форматтар үчүн "fallback" (резервдик вариант) калтырууну унутпаңыз.

4.  **Убактылуу жана Туруктуу Абал (Committed vs Temp State):**
    - **Маселе:** Редакторду жапканда (Exit) же жаңыртканда (Refresh) өзгөрүүлөр жөндөн-жөн өчүп кетчү.
    - **Чечим:** `EditorContext` ичинде ар дайым `temp` (өзгөртүлүп жаткан) жана `committed` (акыркы сакталган) абалды бөлүү зарыл. Бул "Undo/Redo" жана "Exit without safe" функцияларынын туура иштешин камсыз кайлат.

5.  **Анимация жана Пропорция (cqi vs px):**
    - **Маселе:** Анимация учурунда (мисалы, Jump) жылаңач пикселдерди (`px`) колдонуу фондун форматына карап чоюлуп кетет.
    - **Чечим:** Анимациянын бардык маанилерин (hover X/Y, idle amplitude) дайыма `cqi` (container-relative) бирдигине нормалдаштыруу керек (`value / 1920 * 100`).

6.  **1:1 Визуалдык Шайкештик (Editor vs Game Parity):**
    - **Маселе:** Коддогу "жашыруун" анимациялар (мисалы, `scale: 1.05`) колдонуучуну адаштырат.
    - **Чечим:** Редактор менен оюндун ортосунда 100% визуалдык окшоштук (WYSIWYG) болушу шарт. Эгер эффект оюнда болсо, ал редактордо да ошондой көрүнүшү керек.
