# Visual Editor "Шкура" — Негизги Спецификация

## Концепция

"Шкура" — бул оюн интерфейсинин үстүнө кийгизиле турган **визуалдык редактор кабаты**. Ал Core Logic'ке (мээге) тийбейт, бирок UI'ды (HTML/CSS) түзмө-түз өзгөртөт жана өзгөрүүлөрдү файлга жазат.

---

## 📂 Layers UI & Grouping (Техникалык Спекс)

Бул бөлүмдө Layers табынын профессионалдык башкаруу механизми сүрөттөлөт.

### 1. Динамикалык Номерлөө (Floor Numbering)
Ар бир элементтин атынын жанында анын глобалдык "этаж" номери көрсөтүлөт.
- **Логика:** `layerOrder` массивиндеги элементин индекси + 1 (`index + 1`).
- **Тартиби:** Тизмеде эң жогору турган элемент (акыркы индекс) эң чоң номерге ээ болот.
- **Дизайн:** Ашыкча `#` белгиси жок, таза алтын түстөгү сандар.

### 2. Rename (✏️) Инструменти
Ар бир элемент жана группа үчүн атайын "Карандаш" иконкасы кошулган.
- **Иштеши:** Баскычты басканда же названияны басканда inline `input` пайда болот.
- **Сактоо:** `Blur` (фокустан чыкканда) же `Enter` басканда `tempNames` (же `groups` аты) автоматтык түрдө жаңыртылат.

### Layers UI & Grouping (Technical Specs)
1.  **Interactive Rows**: Each row in the Layers list is fully interactive (move, visibility, rename).
2.  **Grouping**: `groups` state maps group IDs to `memberIds` list. `isCollapsed` state controls visibility of nested items.
3.  **Section Filtering**: `activeSectionFilter` allows focusing the Layer list on a specific background territory (1, 2... or ALL).
4.  **Territory Detection**: During D&D `onMouseUp`, the cursor's global Y is compared against section boundaries (`100/sectionCount`) to auto-reassign `sectionId` and remap local coordinates.
### 3. Группалоо (Folders)
Бир нече элементти тандап, аларды бир Группага (папкага) бириктирсе болот.
- **Тип:** `EditorGroup { id, name, memberIds: number[], isCollapsed: boolean }`.
- **Интерактивдүүлүк:** Группанын ичиндеги элементтер сырткылар сыяктуу эле толук функционалдуу (eye, pencil, copy, trash).
- **Expansion Fix:** Группанын контейнери `overflow: visible` болушу шарт. Бул тизме жайылганда бардык ички элементтердин толук көрүнүшүн камсыз кылат.
- **Drag & Drop:** Элементтерди группанын ичинде да, глобалдык тизмеде да сүйрөп жылдырууга болот.

## Эрежелер (🛡️ КАТЫРЫЛГАН)

1. **Редактор режими** — `Ctrl+E` же жогорку которгуч менен күйгүзүлөт.
2. **Анимация Блоктору (Black Box)** — Бардык анимациялар `AnimatedContainer.tsx` ичинде корголгон. Аларга тийүүгө тыюу салынат.
3. **Hard Freeze** — Редактор күйгөндө бардык элементтер `duration: 0` менен "катып", 100% стабилдүү болушу шарт.
4. **SAVE'сиз чыксаң** — " Өзгөртүүлөр сакталган жок" деген эскертме чыгат.
5. **Core Logic'ке** — Редактор эч качан тийбейт.

## 🛡️ Катмарларды Башкаруу жана "Айнек Катмар" (Collision)

Колдонуучу айткандай, кээ бир элементтердин "көрүнбөгөн айнек катмары" (hitbox/bounding box) анын визуалдык өлчөмүнөн чоң болуп, жанындагы элементтерди жаап калышы мүмкүн. Бул маселени төмөнкүдөй чечебиз:

### 1. `pointer-events` Техникасы
Бардык иконкалардын контейнерлерине `pointer-events: none` беребиз. Иконканын же hitbox'тун өзүнө гана `pointer-events: auto` коюлат.
- **Натыйжа:** Иконканын айланасындагы бош "айнектен" чычкан өтүп кете берет, ал эми иконканын өзүнө тийгенде гана басылат.

### 2. Hitbox'ту Визуалдаштыруу (Debug View)
Шкура системасында "Hitbox View" деген режим болот. Аны күйгүзгөндө, бардык элементтердин реалдык басыла турган аймактары (айнек катмарлары) жарым тунук түс (мисалы, кызыл) менен көрүнүп турат.
- **Пайдасы:** Кайсы элемент кайсыны жаап жатканын дароо көрүп, `Resize` инструменти менен анын өлчөмүн тактап койсо болот.

### 3. Z-Index Редактору
Ар бир элементтин кабатын (`z-index`) Шкура аркылуу өзгөртсө болот. Эгер эки иконка бири-биринин үстүнө чыгып жатса, кайсынысы жогору турушу керектигин сен өзүң (Toolbar аркылуу) чечесиң.

### 4. Конформдук Hitbox (Физикалык Дал Келүү)
Колдонуучу айткандай, "айнек катмарлар" (hitboxes) элементтин **реалдуу физикалык өлчөмүнө** мүмкүн болушунча дал келиши керек.
- **Эреже:** Hitbox иконканын өзүнөн өтө чоң болбошу шарт. Ал тыкан, иконканын формасына жараша жана анын өлчөмүнө пропорционалдуу болушу керек. Бул "көрүнбөгөн айнектердин" бири-бирин кокустан жаап калышын 90% алдын алат.

## Рендеринг жана Позициялоо Математикасы (Техникалык)

Кийинки иштеп чыгуучулар үчүн эң маанилүү техникалык эрежелер:

### 1. Эки Кабаттуу Контейнер (Wrapper Pattern)
`MapIcon.tsx` ичинде визуалдык катмар экиге бөлүнгөн:
- **Сырткы `div` (Static Wrapper):** Бул катмар позициялоо (`top`, `left`, `width`, `height`) жана баштапкы `transform: translate(-50%, -50%)` үчүн гана жооп берет.
- **Ички `motion.div` (Animation Layer):** Бул жерде ГАНА `framer-motion` анимациялары (`y`, `scale`, `filter`) иштейт. 
- **ЭМНЕ ҮЧҮН?** `framer-motion` анимация учурунда компоненттин баштапкы CSS `transform` касиеттерин өчүрүп салат. Ошондуктан биз аларды эки башка контейнерге бөлүп, бири-бирине тоскоол болбошун камсыз кылдык.

### 2. `isFullLayer` vs `isPortal` Логикасы
- **`isFullLayer` (ID 1-5):** Булар ар дайым `100vw/100vh` өлчөмүндө жана `0:0` (жогорку-сол) чекитинде турат. Сүрөт алмаштырылса дагы бул статус өзгөрбөшү керек.
- **`isPortal` (ID 6+ же башка):** Бул кадимки иконкалар же тексттер. Алар өздөрүнүн `auto` өлчөмүнө же редактордо берилген өлчөмгө таянышат.

### 3. `transformOrigin` Эрежеси
- **Background Layers:** `transformOrigin` дайыма `currentHandle.x / currentHandle.y` (пайыз менен) болушу шарт. Бул анимацияны экрандын каалаган чекитинен баштоого мүмкүндүк берет.
- **Custom Icons & Texts:** `transformOrigin` дайыма `'center'` болушу керек. Анткени алар кичинекей жана так ортосунан чоңоюп-кичейиши шарт.

### 4. Редактордон Чыгуу (Snapped Mode)
- Редактордон чыкканда (`isSnapped`), элементтер `width/height: 100%` режимине кайтат (эгер алар толук контейнерди толтурушу керек болсо). Бирок кастом тексттер жана сүрөттөр өздөрүнүн туура пропорциясын сактоо үчүн `width: auto` маанисин сактап калышы мүмкүн. Бул математиканы `MapIcon.tsx` ичиндеги `visualWidth/Height` өзгөрмөлөрү башкарат.
