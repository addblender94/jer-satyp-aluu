# 📐 COORDINATE LAWS — Координаталык Системанын Мыйзамдары

Бул документ Шкура редакторундагы элементтердин координаталары, катмарлары жана алардын өз ара байланышы боюнча негизги эрежелерди бекитет. Бул системаны өзгөртүүдө төмөнкү "физика мыйзамдарын" сактоо МИЛДЕТТҮҮ.

---

## 1. Үч Катмардуу Көз Карандысыздык (The 3-Layer Rule)

Ар бир иконка же текст үч параллелдүү катмардан турат:
1.  **Hitbox (Көгүлтүр):** Логикалык талаа (оюнчу каякты басат).
2.  **Visual Layer (Сүрөт/Текст):** Сүрөттүн өзү каякта көрүнөт.
3.  **Selection Overlay (Рамка):** Редактордогу жашыл рамка.

### ⚓ Мыйзам:
*   **ELEMENT режиминде**: Үчөө тең бир бүтүн катары синхрондуу жылат.
*   **HITBOX режиминде**: Сүрөт ордунда калат, рамка менен Hitbox гана ажырап жылат.
*   **VISUAL режиминде**: Hitbox ордунда калат, Сүрөт менен анын туткасы (Pink Frame) гана ажырап жылат.

---

## 2. Кыймыл Синхронизациясы (Movement Synchronization)

ELEMENT режиминде кыймыл (Drag) төмөнкү формулага баш ийет:
`Final_Position = Initial_Position + Mouse_Delta`

### ⚠️ МААНИЛҮҮ ЭРЕЖЕЛЕР:
1.  **Static Start Position**: Кыймыл башталганда (`onMouseDown`), баштапкы координаталар өзгөрбөй сакталышы керек. Дельта (`dx, dy`) ушул туруктуу чекитке гана кошулат. Эгер баштапкы чекитти кыймыл учурунда жаңыртып ийсеңиз, кыймыл "секирип" же "инверсия" болуп бузулат.
2.  **60 FPS Manual Sync**: Кыймыл өтө жумшак болушу үчүн, `onMouseMove` учурунда React State колдонулбайт. Анын ордуна түз DOM аркылуу (`style.top`, `style.left`) координаталар жаңыртылат.
3.  **State Commitment**: Кыймыл бүткөндө гана (`onMouseUp`), акыркы координаталар глобалдык State'ке (Context) жазылат.

## 3. "Full Layer" (Фон/Портал) Архитектурасы

Сүрөтү бар, бирок тексти жок элементтер `isFullLayer` деп аталат.

1.  **Sheet (Фон)**: Эгер өлчөмү `100%` болсо, ал картанын фону катары эсептелет. Ал дайыма `(0,0)` чекитинде катырылып турушу керек.
2.  **Pin/Marker (Иконка)**: Булар да `isFullLayer` болушу мүмкүн, бирок алар эркин жылышы керек. Эч качан аларды `(0,0)` чекитине "бекитип" таштабаңыз.
3.  **Default 0.0**: Бардык иконкалар баштапкы абалында `visualX: 0.00%, visualY: 0.00%` (картанын башы) чекитинде турушу мүмкүн. Бул — "Заводской" абалы. Бирок аларды жылдырууга тыюу салынбашы керек.

## 4. Динамикалык Секциялар жана Пропорция (Dynamic Height Laws)

Картанын бийиктиги дайыма фондук сүрөттүн пропорциясына көз каранды.

1.  **Background-Driven Height**: Секциянын бийиктиги ичиндеги фондун (`isFullLayer` жана `width: 100%`) табигый бийиктигине жараша автоматтык түрдө аныкталат.
2.  **Fixed 1920 Base**: Горизонталдык туурасы дайыма туруктуу (негизи 1920 пиксель). Бул "Туруктуу Канвас" (Stable Canvas) принциби.
3.  **Scaling % Coordinates**: Бардык координаталар процент менен берилгендиктен, карта узарган сайын алар ошол жаңы бийиктикке жараша пропорционалдуу түрдө кайра эсептелет. Элементтер картадан чыгып кетпейт, алар дайыма фондун ичинде калат.
4.  **Aspect Ratio Security**: Сүрөттөр "чоюлбашы" керек (`distort`). Алар өзүнүн табигый пропорциясын сактоо менен секцияны чоңойтуп же кичирейтип турат.

---

## 5. Маалыматтардын тазалыгы (Data Integrity)

1.  **Stale Data Warning**: Кыймыл башталганда дайыма `committedLocations` маалыматын колдонуңуз. Эски `LOCATIONS` дайыма эле актуалдуу болбошу мүмкүн.
2.  **Default Fallbacks**: Эгер элементтин баштапкы координаталары жок болсо:
    *   Hitbox үчүн: `0`
    *   Visual (Иконка) үчүн: `0%`
    *   Visual (Текст) үчүн: `50%` (борборлоштуруу үчүн)

## 6. Келечектеги Ассистенттерге КӨРСӨТМӨ:

Бул системаны оңдоодо же жаңы функция кошууда:
*   Бир катмарды (мисалы, Visual) оңдоп жатып, экинчи катмардын (Hitbox) синхрондуулугун бузбаңыз.
*   Дайыма ELEMENT режиминде баары чогуу жылып жатканын текшериңиз.
*   **Persistence**: Өзгөртүүдөн кийин Сактоо (Save) баскычын басып, кайра жүктөгөндө баары ордунда жатканын текшериңиз.

> [!IMPORTANT]
> Бул мыйзамдарды бузуу — редактордун "физикасынын" бузулушуна жана маалыматтардын жоголушуна алып келет.
